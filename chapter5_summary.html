บทที่ 5

สรุปผลการปฏิบัติงาน

หลังจากพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (A Comprehensive Intelligent Examination Management System) หรือ Smart Exam เสร็จสิ้นแล้ว และได้ทดสอบการทำงานของระบบจากการประเมินความพึงพอใจในการใช้งาน ปัญหาที่พบ และข้อเสนอแนะในการพัฒนาระบบต่อในอนาคตได้ ดังนี้

1.  ผลการปฏิบัติงาน

การพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) ได้จำแนกผลการดำเนินงานออกเป็น 2 ส่วน โดยมีรายละเอียดต่อไปนี้

1)  ผลการพัฒนาระบบ

ผลการพัฒนาระบบสามารถแบ่งออกเป็น 2 ส่วน ได้แก่ ขอบเขตของระบบงานและขอบเขตของผู้ใช้งาน โดยมีรายละเอียดดังต่อไปนี้ ขอบเขตของระบบงานที่สามารถดำเนินการได้ครอบคลุมมีดังนี้

ผลการดำเนินงาน

ขอบเขต ความสำเร็จ

1. การจัดการสิทธิ์ผู้ใช้งาน

  • ผู้ดูแลระบบ (Admin) ระบบสามารถจัดการบัญชีผู้ใช้ทั้งหมด กำหนดสิทธิ์บทบาท ตรวจสอบสถิติภาพรวม และแจ้งเตือนกรณีทุจริตการสอบได้
  • อาจารย์ (Teacher) ระบบสามารถสร้างและจัดการชุดข้อสอบ วิเคราะห์ผลการสอบ และตรวจข้ออัตนัยได้
  • นักศึกษา (Student) ระบบสามารถเข้าสอบด้วย PIN กรอกคำตอบ และดูผลการสอบของตนเองได้
 100%

2. ส่วนของการจัดการข้อมูล

ได้แก่ ข้อมูลผู้ใช้งาน (users) ข้อมูลรายวิชา (courses) ข้อมูลการมอบหมายอาจารย์ผู้สอน (teacher_courses) ข้อมูลชุดข้อสอบ (exams) ข้อมูลคลังโจทย์ (questions) ข้อมูลโจทย์ในชุดสอบ (exam_questions) ข้อมูลผลการสอบ (student_results) และข้อมูลคำตอบนักศึกษา (student_answers)

 100%

3. ส่วนลงชื่อเข้าใช้งานระบบ

  • ผู้ใช้งานทุกบทบาท เข้าสู่ระบบด้วย Username/Password ผ่านระบบ Session Authentication ที่ตรวจสอบ Role และสถานะบัญชีก่อนอนุญาต
  • นักศึกษา เข้าสู่ห้องสอบเพิ่มเติมด้วยรหัส PIN 6 หลักที่อาจารย์กำหนด
 100%

4. ส่วนของการจัดการการสอบ

4.1) สิทธิ์ผู้ดูแลระบบ (Admin)

  • ฟังก์ชันจัดการผู้ใช้งาน พัฒนาระบบสำหรับเพิ่ม แก้ไข ระงับบัญชี นำเข้า/ส่งออกข้อมูลผ่าน CSV และ Reset รหัสผ่านแบบ AJAX
  • ฟังก์ชันจัดการรายวิชาและผู้สอน พัฒนาระบบเพิ่มรายวิชา กำหนดอาจารย์ผู้สอนประจำวิชาผ่าน Multi-select และนำเข้า/ส่งออก CSV
  • แดชบอร์ดสถิติ พัฒนาหน้าจอแสดงภาพรวมจำนวนนักศึกษา อาจารย์ รายวิชา และแจ้งเตือนทุจริตการสอบ พร้อม Line Chart สถิติ 7 วัน

4.2) สิทธิ์อาจารย์ผู้สอน (Teacher)

  • ฟังก์ชันสร้างชุดข้อสอบ พัฒนาระบบสำหรับสร้างการสอบพร้อมกำหนด PIN เวลาสอบ 2 รูปแบบ (ช่วงเวลา/ระยะเวลาอิสระ) ห้องสอบ กลุ่มเรียน และตั้งค่าการสลับข้อสอบ
  • ฟังก์ชันจัดการโจทย์ข้อสอบ พัฒนาระบบเพิ่มโจทย์ 3 รูปแบบ ได้แก่ ปรนัย ถูก-ผิด และอัตนัย พร้อมระบบดึงโจทย์จากคลังแบบสุ่มและกำหนดจำนวน
  • ฟังก์ชันดูผลและตรวจข้อสอบ พัฒนาหน้าแสดงผลการสอบ สถิติคะแนน และระบบตรวจให้คะแนนข้ออัตนัยพร้อม Update คะแนนรวมทันที
  • แดชบอร์ดอาจารย์ พัฒนาหน้าหลักแสดงสถิติ Bar Chart คะแนนสูงสุด/เฉลี่ย/ต่ำสุด และตารางรายการส่งข้อสอบล่าสุด
 100%

5. ส่วนระบบทำข้อสอบออนไลน์สำหรับนักศึกษา

ได้แก่ ระบบนาฬิกาจับเวลาแบบ Real-time ระบบส่งข้อสอบอัตโนมัติ (Auto-submit) เมื่อหมดเวลา ระบบ Auto-save คำตอบใน LocalStorage ป้องกันสูญหายเมื่อหน้าเว็บรีเฟรช และระบบตรวจสอบการทำซ้ำป้องกันไม่ให้ทำข้อสอบชุดเดิมมากกว่าหนึ่งครั้ง

 100%

6. ส่วนการสร้างข้อสอบด้วยปัญญาประดิษฐ์ (AI Generator)

ฟีเจอร์สร้างโจทย์ข้อสอบปรนัยอัตโนมัติโดย Google Gemini API (gemini-2.5-flash-lite) จากการวิเคราะห์เนื้อหาที่อัปโหลดในรูปแบบ PDF (ผ่าน Smalot PDF Parser) หรือข้อความที่ป้อน พร้อมระบบตรวจสอบและปรับแก้ผ่าน AI ก่อนบันทึกลงระบบ

 100%

7. ส่วนการวิเคราะห์คุณภาพข้อสอบเชิงสถิติ (Item Analysis)

ระบบคำนวณค่าความยาก (p) และค่าอำนาจจำแนก (D) ของข้อสอบปรนัยและถูก-ผิดแต่ละข้อ โดยแบ่งกลุ่มผู้สอบเป็นกลุ่มเก่ง (Upper) และกลุ่มอ่อน (Lower) พร้อมข้อเสนอแนะจากระบบ AI อัตโนมัติ

 100%

2.  สรุปผลการดำเนินงาน

จากผลการพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) สามารถใช้งานได้จริงครอบคลุมตามขอบเขตของโครงงาน โดยมีรายละเอียดดังนี้

1)  ผลการทดสอบระบบ

การทดสอบการทำงานของระบบ Smart Exam ได้ดำเนินการทดสอบฟังก์ชันการทำงานหลัก เพื่อตรวจสอบความถูกต้อง แม่นยำ และประสิทธิภาพในการตอบสนองของระบบในสถานการณ์จำลองต่าง ๆ โดยมุ่งเน้นการทดสอบในส่วนของกระบวนการเข้าสอบ การสร้างข้อสอบด้วย AI การตรวจข้อสอบอัตโนมัติ และระบบ Item Analysis ซึ่งผลการทดสอบสามารถแสดงรายละเอียดได้ดังตารางที่ 5.3

ตารางที่ 5.3  ผลการทดสอบฟังก์ชันการทำงานหลักของระบบ

ลำดับ กรณีทดสอบ ขั้นตอนการทดสอบ ผลลัพธ์ที่คาดหวัง ผลการทดสอบ

2)  การพัฒนาระบบ

ในการพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) สามารถพัฒนาระบบได้ครอบคลุมตามวัตถุประสงค์และขอบเขตของโครงงาน ทั้งในส่วนของระบบสำหรับอาจารย์ผู้สอน ระบบสำหรับผู้ดูแลระบบ และระบบทำข้อสอบออนไลน์สำหรับนักศึกษา

จากผลการประเมินในภาพรวมของโครงงานทั้งหมด สะท้อนให้เห็นว่าระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) ที่พัฒนาขึ้นสามารถตอบโจทย์ด้านประสิทธิภาพการจัดการการสอบและความสะดวกสบายในการใช้งานได้อย่างครบวงจร โดยมีผลการประเมินความพึงพอใจเฉลี่ยรวมอยู่ที่ 4.56 ซึ่งอยู่ในเกณฑ์ระดับดีมาก ทั้งในส่วนของระบบสำหรับอาจารย์ผู้สอนที่สามารถสร้างข้อสอบ ใช้ AI วิเคราะห์ข้อสอบจาก PDF และวิเคราะห์คุณภาพข้อสอบเชิงสถิติได้อย่างรวดเร็วและแม่นยำ ควบคู่ไปกับระบบทำข้อสอบออนไลน์สำหรับนักศึกษาที่สามารถเข้าสอบด้วย PIN กรอกคำตอบ และดูผลคะแนนได้อย่างง่ายดาย ซึ่งองค์ประกอบทั้งหมดนี้ช่วยให้กระบวนการสอบมีประสิทธิภาพสูงสุด

3.  ข้อจำกัดของโครงงาน

การพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) ที่พัฒนาขึ้น มีข้อจำกัดที่ควรทราบดังต่อไปนี้

ด้านระบบสำหรับอาจารย์และผู้ดูแลระบบ

1) ด้านการใช้งาน AI Generator ฟีเจอร์สร้างข้อสอบด้วย Google Gemini AI จำเป็นต้องอาศัยการเชื่อมต่ออินเทอร์เน็ตที่มีความเสถียรเพื่อเรียกใช้งาน API หากการเชื่อมต่อขัดข้องหรือ API Key หมดโควต้าจะไม่สามารถใช้งานฟีเจอร์ดังกล่าวได้ และระบบในปัจจุบันสามารถสร้างได้เฉพาะข้อสอบประเภทปรนัย (Multiple Choice) เท่านั้น ยังไม่รองรับการสร้างข้อสอบประเภทถูก-ผิดหรืออัตนัยด้วย AI

2) ด้านการวิเคราะห์ Item Analysis ฟีเจอร์วิเคราะห์คุณภาพข้อสอบสามารถวิเคราะห์ได้เฉพาะข้อสอบประเภทปรนัยและถูก-ผิดเท่านั้น ไม่สามารถวิเคราะห์ข้ออัตนัยได้ เนื่องจากการวิเคราะห์แบบ Dichotomous ไม่เหมาะสมกับคะแนนต่อเนื่อง และต้องมีผู้เข้าสอบอย่างน้อย 2 คนขึ้นไปจึงจะสามารถคำนวณได้

3) ด้านการจัดการไฟล์ PDF ระบบรองรับการอัปโหลด PDF ที่มีข้อความแบบ Text-based เท่านั้น ไม่รองรับ PDF ที่เป็นไฟล์รูปภาพสแกน (Scanned PDF) ซึ่งจะต้องผ่านกระบวนการ OCR ก่อน

ด้านระบบทำข้อสอบสำหรับนักศึกษา

1) ด้านความต้องการของระบบ ระบบรองรับเฉพาะการใช้งานผ่านเว็บเบราว์เซอร์บนอุปกรณ์ที่มีการเชื่อมต่ออินเทอร์เน็ต ยังไม่มีแอปพลิเคชันบนมือถือ (Mobile App) และยังไม่รองรับการทำข้อสอบแบบออฟไลน์

2) ด้านการป้องกันการทุจริต ระบบในปัจจุบันสามารถตรวจจับพฤติกรรมน่าสงสัยได้เพียงบางส่วน เช่น เวลาทำข้อสอบสั้นผิดปกติ แต่ยังไม่มีระบบตรวจจับการออกจากหน้าเว็บ (Tab Switching) หรือการจับภาพหน้าจอ

4.  ข้อเสนอแนะเพื่อการพัฒนาต่อ

การพัฒนาระบบบริหารจัดการการสอบอัจฉริยะแบบครบวงจร (Smart Exam) ในอนาคตสามารถพัฒนาต่อยอดได้ดังต่อไปนี้

ด้านการพัฒนาระบบสำหรับอาจารย์และผู้ดูแลระบบ

1) ขยายความสามารถของ AI Generator พัฒนาให้ AI สามารถสร้างข้อสอบหลายรูปแบบได้พร้อมกัน ทั้งปรนัย ถูก-ผิด และอัตนัย พร้อมเพิ่มความสามารถในการสรุปเนื้อหาจากไฟล์ PDF แบบ Scanned ด้วยการนำเทคโนโลยี OCR มาบูรณาการ

2) พัฒนาระบบรายงานเชิงลึก ควรมีการพัฒนาหน้ารายงานให้สามารถแสดงผลเปรียบเทียบสถิติคะแนนระหว่างกลุ่มเรียน ระหว่างชุดข้อสอบ และระหว่างเทอม รวมถึงรองรับการ Export รายงานในรูปแบบ PDF และ Excel

3) พัฒนาระบบจัดการห้องสอบ (Proctoring) ควรมีการเพิ่มฟีเจอร์ตรวจจับการออกจากหน้าเว็บ (Tab Switching Detection) และบันทึก Log การกระทำของนักศึกษาระหว่างสอบ เพื่อเพิ่มความน่าเชื่อถือของกระบวนการสอบ

ด้านการพัฒนาระบบสำหรับนักศึกษา

1) พัฒนา Mobile Application พัฒนาแอปพลิเคชันบนมือถือทั้ง iOS และ Android เพื่อรองรับการทำข้อสอบบนสมาร์ตโฟนได้สะดวกยิ่งขึ้น และรองรับการทำข้อสอบแบบออฟไลน์โดยซิงค์ข้อมูลเมื่อมีอินเทอร์เน็ต

2) การกรองและแนะนำเนื้อหาเฉพาะบุคคล พัฒนาให้ระบบสามารถวิเคราะห์จุดอ่อนของนักศึกษาจากประวัติการสอบ และแนะนำข้อสอบที่เหมาะสมกับระดับความสามารถของแต่ละคน เพื่อส่งเสริมการเรียนรู้แบบ Adaptive Learning

3) การรองรับหลายภาษา เนื่องจากสถาบันการศึกษามีนักศึกษาต่างชาติ ควรมีการนำ AI มาประยุกต์ใช้ในการแปลภาษาข้อสอบอัตโนมัติ ตามการตั้งค่าภาษาของผู้ใช้งาน เพื่อให้ระบบสามารถรองรับผู้ใช้ได้หลากหลายกลุ่มมากยิ่งขึ้น
